# AI Vibe Workflow - 제품 개요

## 📌 한줄 소개
**AI 개발자를 위한 3단계 워크플로우 자동화 플랫폼**

## 🎯 제품 비전

AI 개발자들이 반복적인 설정 작업에 시간을 낭비하지 않고, 핵심 가치인 **프롬프트 개발과 AI 활용**에만 집중할 수 있도록 돕는 자동화 플랫폼입니다.

## 💼 타겟 사용자

### Primary Users
- **AI 프롬프트 엔지니어**: 여러 프로젝트에서 프롬프트를 재사용하고 개선
- **AI 앱 개발자**: 반복적인 프로젝트 설정을 자동화하고 싶은 개발자
- **스타트업 CTO**: 빠른 프로토타이핑이 필요한 기술 리더

### Secondary Users
- **프리랜서 개발자**: 여러 클라이언트 프로젝트의 설정을 표준화
- **개발팀 리더**: 팀 전체의 워크플로우를 통일하고 생산성 향상
- **AI 학습자**: AI 개발 베스트 프랙티스를 배우고 적용

## 🚀 3단계 워크플로우 상세

### Stage 1: Intro (프로젝트 초기화)
**목적**: 반복적인 프로젝트 설정 작업 자동화

**주요 기능**:
- 🔄 기존 프로젝트에서 프롬프트와 설정 재사용
- 📦 GitHub 저장소 자동 생성 및 초기 커밋
- 🎨 프로젝트 템플릿 라이브러리
- 🔍 Vector 검색으로 유사 프로젝트 찾기

**시간 절감**:
- 기존: 30분 (수동 설정, 파일 복사, 초기화)
- AI Vibe: 5분 (자동화된 워크플로우)
- **절감률: 83%**

### Stage 2: Contents (프롬프트 개발)
**목적**: AI 프롬프트 개발과 최적화에 집중

**주요 기능**:
- 🤖 멀티 AI 모델 동시 비교 (GPT-4, Claude, Gemini)
- 📝 고급 프롬프트 에디터 (Monaco Editor with AI)
- 📊 응답 품질 분석 및 비교
- 💾 프롬프트 버전 관리
- 🔖 프롬프트 라이브러리 및 태깅

**AI 크레딧 절감**:
- 최적화된 프롬프트로 재시도 감소
- 효과적인 모델 선택
- **절감률: 30%**

### Stage 3: Deploy (배포 자동화)
**목적**: 클라우드 배포와 인프라 설정 자동화

**주요 기능**:
- ☁️ Vercel/Netlify 원클릭 배포
- 🗄️ Supabase PostgreSQL 자동 설정
- 🔐 환경 변수 자동 구성
- 🌐 도메인 및 SSL 자동 설정
- 📊 배포 상태 모니터링

**설정 시간 절감**:
- 기존: 45분 (수동 배포, DB 설정, 환경 변수)
- AI Vibe: 7분 (자동화된 배포)
- **절감률: 84%**

## 📊 핵심 성과 지표 (KPI)

- ♻️ **자산 재사용률 70% 달성**: 7/10 프로젝트에서 기존 자산 활용
- 💰 **AI 크레딧 30% 절감**: 월 $100 → $70

### 품질 지표
- 📈 프롬프트 품질 향상
- 🐛 배포 에러 50% 감소
- 🔄 반복 작업 자동화율 80%

## 🏗️ 기술 아키텍처

### Frontend
```
Next.js 15 + React Flow + Zustand + TypeScript
- 시각적 워크플로우 빌더 (React Flow)
- 고급 코드 에디터 (Monaco Editor)
- 반응형 UI (Tailwind CSS)
```

### Backend
```
Fastify + Zod + Prisma + PostgreSQL
- RESTful API (Fastify)
- 타입 안전성 (Zod validation)
- 데이터베이스 (Prisma ORM)
```

### AI & Integration
```
- OpenAI GPT-4 API
- Anthropic Claude API
- Google Gemini API
- GitHub API (저장소 관리)
- Vercel API (배포 자동화)
```

### Database
```
PostgreSQL with Supabase
- pgvector for semantic search
- 프롬프트 버전 관리
- 사용자 프로필 및 프로젝트 메타데이터
```

## 🎨 UI/UX 디자인 철학

### 시각적 워크플로우
- 블록 기반 인터페이스 (React Flow)
- 드래그 앤 드롭으로 워크플로우 구성
- 실시간 프리뷰

### 색상 시스템
- 🚀 **Intro**: 파란색 계열 (신뢰, 시작)
- ✏️ **Contents**: 초록색 계열 (성장, 창작)
- 🔧 **Deploy**: 주황색 계열 (에너지, 완성)

### 사용성 원칙
1. **간단함**: 3단계로 명확히 구분
2. **시각화**: 워크플로우를 한눈에 파악
3. **자동화**: 최소한의 입력으로 최대 효과
4. **투명성**: 각 단계의 진행 상황 실시간 표시

## 🔄 사용자 여정 (User Journey)

### 첫 방문
1. 랜딩 페이지에서 3단계 워크플로우 확인
2. "워크플로우 빌더 시작하기" 클릭
3. GitHub로 로그인
4. 빠른 튜토리얼 (2분)

### 프로젝트 생성
1. **Intro**: 기존 프로젝트 선택 또는 템플릿 사용
2. **Contents**: 프롬프트 작성 및 AI 모델 테스트
3. **Deploy**: 원클릭 배포

### 일상적 사용
1. 대시보드에서 프로젝트 목록 확인
2. 프롬프트 라이브러리에서 재사용
3. 새 프로젝트에 기존 설정 적용
4. 팀원과 프롬프트 공유
